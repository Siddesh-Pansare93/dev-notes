# Recommended Reading

> [!info] For the Express/TS dev
> A curated list of canonical resources. Names only — search the title to find the official source. The Java/Spring world has excellent free official docs; lean on those before chasing tutorials.

## Official documentation (always start here)

- **Spring Boot Reference Documentation** — the single most useful resource
- **Spring Framework Reference Documentation** — for core IoC, AOP, transactions
- **Spring Security Reference Documentation**
- **Spring Data JPA Reference Documentation**
- **Java Language Specification (JLS)** — for the pedantic
- **Java API Documentation (Javadoc)** — JDK class library reference
- **OpenJDK Project Documentation**

## Books — Java language

- **Effective Java** by Joshua Bloch — *the* book on writing idiomatic Java. Read 2-3 items per week.
- **Java Concurrency in Practice** by Brian Goetz et al. — concurrency bible
- **Modern Java in Action** by Raoul-Gabriel Urma, Mario Fusco, Alan Mycroft — streams, lambdas, modules
- **The Well-Grounded Java Developer (2nd ed.)** by Benjamin J. Evans, Jason Clark, Martijn Verburg
- **Core Java, Volumes I & II** by Cay Horstmann — comprehensive language reference
- **Java Performance: The Definitive Guide** by Scott Oaks

## Books — Spring & Spring Boot

- **Spring in Action** by Craig Walls — broad Spring Boot coverage
- **Spring Boot in Practice** by Somnath Musib
- **Spring Microservices in Action** by John Carnell
- **Pro Spring Boot 3** by Felipe Gutierrez

## Books — architecture & design

- **Domain-Driven Design** by Eric Evans (the Blue Book)
- **Implementing Domain-Driven Design** by Vaughn Vernon (the Red Book)
- **Patterns of Enterprise Application Architecture** by Martin Fowler
- **Building Microservices (2nd ed.)** by Sam Newman
- **Designing Data-Intensive Applications** by Martin Kleppmann — language-agnostic, essential
- **Release It!** by Michael Nygard — production patterns (timeouts, circuit breakers, bulkheads)

## Books — testing

- **Growing Object-Oriented Software, Guided by Tests** by Steve Freeman, Nat Pryce
- **Unit Testing Principles, Practices, and Patterns** by Vladimir Khorikov
- **xUnit Test Patterns** by Gerard Meszaros

## Books — JVM & performance

- **Optimizing Java** by Benjamin J. Evans, James Gough, Chris Newland
- **The Garbage Collection Handbook** by Richard Jones, Antony Hosking, Eliot Moss

## Online courses

- **Spring Academy** (official, free tier) — by Broadcom/VMware. Spring Certified Professional path.
- **Baeldung Learn Spring Series**
- **Java Brains** (YouTube) — by Koushik Kothagal
- **Marco Codes** (YouTube) — modern Spring Boot 3 content
- **Amigoscode** (YouTube)
- **Dan Vega** (YouTube + Spring Developer Advocate)

## Blogs and reference sites

- **Baeldung** — exhaustive how-tos. Use as a reference, not a tutorial sequence.
- **Spring Blog** (official) — release announcements, deep-dives
- **InfoQ — Java track**
- **Foojay.io** — Java community hub
- **Vlad Mihalcea's blog** — JPA/Hibernate deep expertise
- **Thorben Janssen's blog** — JPA/Hibernate
- **Reflectoring.io** — solid Spring Boot articles

## Specs (when you need to go deep)

- **Jakarta EE Specifications** — Servlet, JPA, Bean Validation, JSON-B, etc.
- **JEPs (JDK Enhancement Proposals)** — language feature roadmap
- **W3C Trace Context** — distributed tracing propagation
- **OpenAPI Specification**
- **OAuth 2.1 / OIDC Core 1.0**

## Reference architectures

- **Spring PetClinic** — canonical Spring Boot sample (and its microservices/k8s/native variants)
- **Spring Boot samples** in the spring-projects GitHub org

## How to use this list

> [!tip] Reading strategy
> 1. Read **Effective Java** in parallel with daily coding — even if slow
> 2. Bookmark the **Spring Boot Reference Documentation** and reach for it before Stack Overflow
> 3. Pick **one** architecture book per quarter (DDIA → Building Microservices → DDD)
> 4. Subscribe to Java Weekly + one YouTube channel for passive osmosis
