# Default Values and Generated Columns

## Theory

PostgreSQL provides powerful features for automatically populating column values without explicit user input. These mechanisms reduce application code complexity, ensure consistency, and prevent common data entry errors.

**Default Values** are expressions evaluated when a row is inserted without providing a value for that column. Defaults can be static values, function calls, or expressions. They're particularly useful for timestamps, unique identifiers, and status flags.

**Generated Columns** are computed from other columns in the same row. PostgreSQL supports GENERATED ALWAYS AS (stored), where values are calculated and physically stored in the table. Generated columns are ideal for derived values, normalized data, and maintaining computed fields.

**Identity Columns** provide SQL-standard auto-incrementing behavior. PostgreSQL offers two variants:
- GENERATED ALWAYS AS IDENTITY: Strictly enforces automatic generation
- GENERATED BY DEFAULT AS IDENTITY: Allows manual override

**SERIAL** is PostgreSQL's legacy auto-increment mechanism, still widely used but less flexible than identity columns.

Understanding these features helps you design tables that enforce business rules, maintain data consistency, and reduce application complexity.

## Syntax

### DEFAULT Values

```sql
CREATE TABLE table_name (
    column_name data_type DEFAULT default_expression
);

-- Add default to existing column
ALTER TABLE table_name ALTER COLUMN column_name SET DEFAULT default_expression;

-- Remove default
ALTER TABLE table_name ALTER COLUMN column_name DROP DEFAULT;
```

### GENERATED ALWAYS AS (Stored Computed Columns)

```sql
CREATE TABLE table_name (
    column1 data_type,
    column2 data_type,
    computed_column data_type GENERATED ALWAYS AS (expression) STORED
);
```

### GENERATED ALWAYS AS IDENTITY

```sql
CREATE TABLE table_name (
    id INTEGER GENERATED ALWAYS AS IDENTITY PRIMARY KEY
);

-- With options
CREATE TABLE table_name (
    id INTEGER GENERATED ALWAYS AS IDENTITY (
        START WITH 1000
        INCREMENT BY 1
        MINVALUE 1000
        MAXVALUE 9999
        CACHE 10
    ) PRIMARY KEY
);
```

### GENERATED BY DEFAULT AS IDENTITY

```sql
CREATE TABLE table_name (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
);
```

### SERIAL (Legacy)

```sql
CREATE TABLE table_name (
    id SERIAL PRIMARY KEY
);

-- Equivalent to:
CREATE SEQUENCE table_name_id_seq;
CREATE TABLE table_name (
    id INTEGER NOT NULL DEFAULT nextval('table_name_id_seq') PRIMARY KEY
);
ALTER SEQUENCE table_name_id_seq OWNED BY table_name.id;
```

## Examples

### Static DEFAULT Values

```sql
-- Basic static defaults
CREATE TABLE products (
    product_id SERIAL PRIMARY KEY,
    product_name VARCHAR(200) NOT NULL,
    status VARCHAR(20) DEFAULT 'active',
    stock_quantity INTEGER DEFAULT 0,
    is_featured BOOLEAN DEFAULT false,
    weight_kg NUMERIC(8, 2) DEFAULT 0.00
);

-- Insert without specifying default columns
INSERT INTO products (product_name) VALUES ('Laptop');

-- Insert with some defaults overridden
INSERT INTO products (product_name, status, stock_quantity)
VALUES ('Tablet', 'pending', 50);

-- Insert explicitly using DEFAULT keyword
INSERT INTO products (product_name, status) VALUES ('Phone', DEFAULT);

SELECT * FROM products;
```

### Function-based DEFAULT Values

```sql
-- Common function defaults
CREATE TABLE events (
    event_id SERIAL PRIMARY KEY,
    event_name VARCHAR(200) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_date DATE DEFAULT CURRENT_DATE,
    event_start TIMESTAMP DEFAULT now(),
    timezone VARCHAR(50) DEFAULT current_setting('TIMEZONE')
);

-- UUID default (requires pgcrypto extension)
CREATE EXTENSION IF NOT EXISTS pgcrypto;

CREATE TABLE users (
    user_id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    created_at TIMESTAMP DEFAULT now(),
    session_token UUID DEFAULT gen_random_uuid()
);

-- Insert data
INSERT INTO users (username) VALUES ('alice'), ('bob');

SELECT * FROM users;

-- Random default values
CREATE TABLE lottery_tickets (
    ticket_id SERIAL PRIMARY KEY,
    ticket_number INTEGER DEFAULT floor(random() * 1000000)::INTEGER,
    purchased_at TIMESTAMP DEFAULT now()
);

INSERT INTO lottery_tickets DEFAULT VALUES;
INSERT INTO lottery_tickets DEFAULT VALUES;
INSERT INTO lottery_tickets DEFAULT VALUES;

SELECT * FROM lottery_tickets;
```

### Adding and Modifying Defaults

```sql
-- Create table without defaults
CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INTEGER NOT NULL,
    order_date DATE,
    status VARCHAR(20),
    notes TEXT
);

-- Add defaults to existing columns
ALTER TABLE orders
ALTER COLUMN order_date SET DEFAULT CURRENT_DATE;

ALTER TABLE orders
ALTER COLUMN status SET DEFAULT 'pending';

-- Insert to test new defaults
INSERT INTO orders (customer_id) VALUES (101);

SELECT * FROM orders;

-- Change default value
ALTER TABLE orders
ALTER COLUMN status SET DEFAULT 'new';

INSERT INTO orders (customer_id) VALUES (102);

-- Remove default
ALTER TABLE orders
ALTER COLUMN status DROP DEFAULT;

-- This will insert NULL for status now
INSERT INTO orders (customer_id) VALUES (103);

SELECT * FROM orders;
```

### GENERATED ALWAYS AS (Stored Computed Columns)

```sql
-- Basic computed column
CREATE TABLE rectangles (
    rectangle_id SERIAL PRIMARY KEY,
    width NUMERIC(10, 2) NOT NULL,
    height NUMERIC(10, 2) NOT NULL,
    area NUMERIC(10, 2) GENERATED ALWAYS AS (width * height) STORED,
    perimeter NUMERIC(10, 2) GENERATED ALWAYS AS (2 * (width + height)) STORED
);

-- Insert data - area and perimeter computed automatically
INSERT INTO rectangles (width, height) VALUES
    (10.5, 20.0),
    (15.0, 15.0),
    (8.25, 12.75);

SELECT * FROM rectangles;

-- Cannot manually insert into generated column
-- This fails:
-- INSERT INTO rectangles (width, height, area) VALUES (10, 20, 200);

-- Update base columns - generated columns update automatically
UPDATE rectangles SET width = 12.0 WHERE rectangle_id = 1;

SELECT * FROM rectangles;
```

### Complex Generated Columns

```sql
-- Full name from first and last name
CREATE TABLE employees (
    employee_id SERIAL PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    full_name VARCHAR(101) GENERATED ALWAYS AS (first_name || ' ' || last_name) STORED,
    email VARCHAR(100),
    email_domain VARCHAR(100) GENERATED ALWAYS AS (
        substring(email from '@(.*)$')
    ) STORED
);

INSERT INTO employees (first_name, last_name, email) VALUES
    ('John', 'Doe', 'john.doe@example.com'),
    ('Jane', 'Smith', 'jane.smith@company.org'),
    ('Bob', 'Johnson', 'bob.j@startup.io');

SELECT employee_id, full_name, email_domain FROM employees;

-- Price calculations
CREATE TABLE invoice_items (
    item_id SERIAL PRIMARY KEY,
    description VARCHAR(200),
    quantity INTEGER NOT NULL CHECK (quantity > 0),
    unit_price NUMERIC(10, 2) NOT NULL CHECK (unit_price > 0),
    subtotal NUMERIC(10, 2) GENERATED ALWAYS AS (quantity * unit_price) STORED,
    tax_rate NUMERIC(4, 3) DEFAULT 0.075,
    tax_amount NUMERIC(10, 2) GENERATED ALWAYS AS (quantity * unit_price * tax_rate) STORED,
    total NUMERIC(10, 2) GENERATED ALWAYS AS (quantity * unit_price * (1 + tax_rate)) STORED
);

INSERT INTO invoice_items (description, quantity, unit_price) VALUES
    ('Widget A', 5, 19.99),
    ('Widget B', 3, 45.50);

SELECT description, quantity, unit_price, subtotal, tax_amount, total
FROM invoice_items;
```

### GENERATED ALWAYS AS IDENTITY

```sql
-- Basic identity column
CREATE TABLE customers (
    customer_id INTEGER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    customer_name VARCHAR(100) NOT NULL,
    email VARCHAR(100)
);

-- Insert data - customer_id generated automatically
INSERT INTO customers (customer_name, email) VALUES
    ('Acme Corp', 'contact@acme.com'),
    ('TechStart Inc', 'info@techstart.com');

SELECT * FROM customers;

-- Cannot override GENERATED ALWAYS
-- This fails:
-- INSERT INTO customers (customer_id, customer_name, email)
-- VALUES (100, 'Manual ID', 'test@test.com');

-- But you can override with OVERRIDING SYSTEM VALUE
INSERT INTO customers (customer_id, customer_name, email)
OVERRIDING SYSTEM VALUE
VALUES (100, 'Manual ID', 'test@test.com');

SELECT * FROM customers;
```

### GENERATED ALWAYS AS IDENTITY with Options

```sql
-- Identity with custom sequence options
CREATE TABLE invoices (
    invoice_id INTEGER GENERATED ALWAYS AS IDENTITY (
        START WITH 10000
        INCREMENT BY 1
        MINVALUE 10000
        MAXVALUE 99999
        CACHE 20
    ) PRIMARY KEY,
    invoice_date DATE DEFAULT CURRENT_DATE,
    customer_id INTEGER NOT NULL,
    total_amount NUMERIC(10, 2)
);

INSERT INTO invoices (customer_id, total_amount) VALUES
    (1, 150.00),
    (2, 275.50),
    (1, 89.99);

SELECT * FROM invoices;

-- Cycle example (for demonstration - usually not recommended for IDs)
CREATE TABLE rotating_schedule (
    slot_id INTEGER GENERATED ALWAYS AS IDENTITY (
        START WITH 1
        INCREMENT BY 1
        MINVALUE 1
        MAXVALUE 7
        CYCLE
    ) PRIMARY KEY,
    day_name VARCHAR(20)
);
```

### GENERATED BY DEFAULT AS IDENTITY

```sql
-- More flexible identity column
CREATE TABLE products_flexible (
    product_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    product_name VARCHAR(200) NOT NULL,
    category VARCHAR(50)
);

-- Auto-generated ID
INSERT INTO products_flexible (product_name, category)
VALUES ('Product A', 'Electronics');

-- Manually specified ID (allowed with BY DEFAULT)
INSERT INTO products_flexible (product_id, product_name, category)
VALUES (1000, 'Product B', 'Books');

-- Auto-generated continues from highest value
INSERT INTO products_flexible (product_name, category)
VALUES ('Product C', 'Toys');

SELECT * FROM products_flexible ORDER BY product_id;

-- Useful for data migration
INSERT INTO products_flexible (product_id, product_name, category) VALUES
    (5000, 'Legacy Product 1', 'Archive'),
    (5001, 'Legacy Product 2', 'Archive');

-- New inserts continue from current max
INSERT INTO products_flexible (product_name, category)
VALUES ('Product D', 'Electronics');

SELECT * FROM products_flexible ORDER BY product_id;
```

### SERIAL vs IDENTITY

```sql
-- Old style: SERIAL
CREATE TABLE orders_serial (
    order_id SERIAL PRIMARY KEY,
    order_date DATE DEFAULT CURRENT_DATE
);

-- Modern style: IDENTITY
CREATE TABLE orders_identity (
    order_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    order_date DATE DEFAULT CURRENT_DATE
);

-- Both work similarly for basic use
INSERT INTO orders_serial (order_date) VALUES (CURRENT_DATE);
INSERT INTO orders_identity (order_date) VALUES (CURRENT_DATE);

SELECT * FROM orders_serial;
SELECT * FROM orders_identity;

-- But SERIAL is actually shorthand for:
CREATE SEQUENCE orders_serial_order_id_seq OWNED BY orders_serial.order_id;
ALTER TABLE orders_serial ALTER COLUMN order_id SET DEFAULT nextval('orders_serial_order_id_seq');

-- View sequences
\ds
```

### Sequence Management

```sql
-- Working with sequences directly
CREATE TABLE tickets (
    ticket_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ticket_number VARCHAR(20),
    created_at TIMESTAMP DEFAULT now()
);

-- Insert some data
INSERT INTO tickets DEFAULT VALUES;
INSERT INTO tickets DEFAULT VALUES;

-- Check current sequence value
SELECT currval(pg_get_serial_sequence('tickets', 'ticket_id'));

-- Set sequence to specific value
ALTER TABLE tickets ALTER COLUMN ticket_id RESTART WITH 1000;

-- Next insert uses new sequence value
INSERT INTO tickets DEFAULT VALUES;
SELECT * FROM tickets;

-- Manually advance sequence
SELECT setval(pg_get_serial_sequence('tickets', 'ticket_id'), 2000);

INSERT INTO tickets DEFAULT VALUES;
SELECT * FROM tickets;

-- Get next value without consuming it
SELECT currval(pg_get_serial_sequence('tickets', 'ticket_id'));

-- Get next value and advance sequence
SELECT nextval(pg_get_serial_sequence('tickets', 'ticket_id'));
```

### Combining Defaults and Generated Columns

```sql
-- Comprehensive example with multiple features
CREATE TABLE sales_transactions (
    -- Identity column
    transaction_id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,

    -- UUID for external reference
    external_id UUID DEFAULT gen_random_uuid() NOT NULL UNIQUE,

    -- Timestamp defaults
    created_at TIMESTAMP DEFAULT now() NOT NULL,
    transaction_date DATE DEFAULT CURRENT_DATE NOT NULL,

    -- Business data
    customer_id INTEGER NOT NULL,
    product_id INTEGER NOT NULL,
    quantity INTEGER NOT NULL CHECK (quantity > 0),
    unit_price NUMERIC(10, 2) NOT NULL CHECK (unit_price > 0),

    -- Generated calculations
    subtotal NUMERIC(10, 2) GENERATED ALWAYS AS (quantity * unit_price) STORED,

    -- Tax calculation with default rate
    tax_rate NUMERIC(5, 4) DEFAULT 0.0825,
    tax_amount NUMERIC(10, 2) GENERATED ALWAYS AS (quantity * unit_price * tax_rate) STORED,

    -- Total
    total_amount NUMERIC(10, 2) GENERATED ALWAYS AS (
        quantity * unit_price * (1 + tax_rate)
    ) STORED,

    -- Status with default
    status VARCHAR(20) DEFAULT 'completed' NOT NULL,

    -- Computed status flag
    is_high_value BOOLEAN GENERATED ALWAYS AS (quantity * unit_price > 1000) STORED
);

-- Insert minimal data, let defaults and generated columns handle the rest
INSERT INTO sales_transactions (customer_id, product_id, quantity, unit_price) VALUES
    (101, 501, 2, 45.99),
    (102, 502, 1, 1299.99),
    (103, 503, 5, 12.50);

SELECT
    transaction_id,
    external_id,
    quantity,
    unit_price,
    subtotal,
    tax_amount,
    total_amount,
    is_high_value,
    status
FROM sales_transactions;
```

### Time-based Defaults

```sql
CREATE TABLE audit_log (
    log_id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,

    -- Different timestamp types
    logged_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    logged_at_tz TIMESTAMP WITH TIME ZONE DEFAULT now(),
    log_date DATE DEFAULT CURRENT_DATE,
    log_time TIME DEFAULT CURRENT_TIME,

    -- Specific timezone
    logged_utc TIMESTAMP DEFAULT (now() AT TIME ZONE 'UTC'),

    -- Future date default (30 days from now)
    expires_at TIMESTAMP DEFAULT (CURRENT_TIMESTAMP + INTERVAL '30 days'),

    action VARCHAR(50),
    details TEXT
);

INSERT INTO audit_log (action, details) VALUES
    ('USER_LOGIN', 'User logged in successfully'),
    ('DATA_EXPORT', 'Exported 1000 records');

SELECT
    log_id,
    logged_at,
    log_date,
    expires_at,
    action
FROM audit_log;
```

## Common Mistakes

### 1. Trying to INSERT into Generated Column

```sql
-- WRONG: Cannot manually insert into generated column
CREATE TABLE wrong_example (
    id SERIAL PRIMARY KEY,
    width NUMERIC,
    height NUMERIC,
    area NUMERIC GENERATED ALWAYS AS (width * height) STORED
);

-- This fails:
-- INSERT INTO wrong_example (width, height, area) VALUES (10, 20, 200);

-- CORRECT: Only insert base columns
INSERT INTO wrong_example (width, height) VALUES (10, 20);
```

### 2. Forgetting DEFAULT Keyword for NULL Override

```sql
CREATE TABLE items (
    item_id SERIAL PRIMARY KEY,
    item_name VARCHAR(100),
    status VARCHAR(20) DEFAULT 'active'
);

-- WRONG: Explicitly passing NULL overrides default
INSERT INTO items (item_name, status) VALUES ('Item 1', NULL);

-- CORRECT: Use DEFAULT keyword or omit column
INSERT INTO items (item_name, status) VALUES ('Item 2', DEFAULT);
INSERT INTO items (item_name) VALUES ('Item 3');

SELECT * FROM items;
```

### 3. Using GENERATED ALWAYS without OVERRIDING SYSTEM VALUE

```sql
CREATE TABLE strict_ids (
    id INTEGER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    name VARCHAR(100)
);

-- WRONG: This fails
-- INSERT INTO strict_ids (id, name) VALUES (100, 'Test');

-- CORRECT: Use OVERRIDING SYSTEM VALUE
INSERT INTO strict_ids (id, name)
OVERRIDING SYSTEM VALUE
VALUES (100, 'Test');

-- Or use GENERATED BY DEFAULT instead
CREATE TABLE flexible_ids (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(100)
);

INSERT INTO flexible_ids (id, name) VALUES (100, 'Test'); -- Works
```

### 4. Not Understanding DEFAULT vs GENERATED

```sql
-- WRONG: Confusing DEFAULT with GENERATED
CREATE TABLE confusion (
    price NUMERIC(10, 2),
    tax NUMERIC(10, 2) DEFAULT price * 0.1  -- This doesn't work!
);

-- CORRECT: Use GENERATED for computed values
CREATE TABLE correct_computation (
    price NUMERIC(10, 2),
    tax NUMERIC(10, 2) GENERATED ALWAYS AS (price * 0.1) STORED
);
```

### 5. Sequence Out of Sync After Manual Inserts

```sql
CREATE TABLE products_seq (
    product_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    product_name VARCHAR(100)
);

-- Manual insert with high ID
INSERT INTO products_seq (product_id, product_name) VALUES (1000, 'Product A');

-- Auto-increment starts from 1, causing conflict
INSERT INTO products_seq (product_name) VALUES ('Product B');
INSERT INTO products_seq (product_name) VALUES ('Product C');

-- CORRECT: Reset sequence after manual inserts
SELECT setval(pg_get_serial_sequence('products_seq', 'product_id'),
              (SELECT MAX(product_id) FROM products_seq));

INSERT INTO products_seq (product_name) VALUES ('Product D');
```

## Best Practices

### 1. Prefer IDENTITY over SERIAL

```sql
-- OLD: SERIAL
CREATE TABLE old_style (
    id SERIAL PRIMARY KEY
);

-- BETTER: IDENTITY (SQL standard, more control)
CREATE TABLE new_style (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
);
```

### 2. Use UUID for Distributed Systems

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;

CREATE TABLE distributed_records (
    record_id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    created_at TIMESTAMP DEFAULT now(),
    data JSONB
);

-- UUIDs prevent conflicts when merging data from multiple sources
```

### 3. Use Generated Columns for Frequently Computed Values

```sql
-- Good use case: avoid repeated calculations in queries
CREATE TABLE customer_orders (
    order_id INTEGER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    customer_id INTEGER,
    subtotal NUMERIC(10, 2),
    tax_rate NUMERIC(5, 4) DEFAULT 0.0825,
    shipping NUMERIC(10, 2) DEFAULT 0,

    -- Store computed total for fast queries
    total NUMERIC(10, 2) GENERATED ALWAYS AS (
        subtotal * (1 + tax_rate) + shipping
    ) STORED
);

-- Can now query and index on total efficiently
CREATE INDEX idx_orders_total ON customer_orders(total);
```

### 4. Set Appropriate Defaults for Status Columns

```sql
CREATE TABLE user_accounts (
    account_id INTEGER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL,
    status VARCHAR(20) DEFAULT 'pending' NOT NULL,
    email_verified BOOLEAN DEFAULT false NOT NULL,
    created_at TIMESTAMP DEFAULT now() NOT NULL,
    last_login TIMESTAMP
);
```

### 5. Document Generated Column Logic

```sql
CREATE TABLE financial_records (
    record_id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    gross_amount NUMERIC(12, 2),
    deductions NUMERIC(12, 2) DEFAULT 0,
    net_amount NUMERIC(12, 2) GENERATED ALWAYS AS (gross_amount - deductions) STORED
);

COMMENT ON COLUMN financial_records.net_amount IS
    'Automatically calculated as gross_amount minus deductions. Cannot be manually set.';
```

### 6. Use Meaningful Sequence Ranges

```sql
-- Different ID ranges for different record types
CREATE TABLE invoice_headers (
    invoice_id INTEGER GENERATED ALWAYS AS IDENTITY (
        START WITH 100000
        INCREMENT BY 1
    ) PRIMARY KEY
);

CREATE TABLE credit_notes (
    credit_note_id INTEGER GENERATED ALWAYS AS IDENTITY (
        START WITH 200000
        INCREMENT BY 1
    ) PRIMARY KEY
);
```
